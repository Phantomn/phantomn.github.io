## 취약점 설명 및 조치 방안
### 취약점 개요
`POST /api/hub/member/certEffecCheck`는 비밀번호 찾기 흐름에서 본인인증 유효 여부를 확인하는 엔드포인트다. 이 엔드포인트는 요청자의 서명 토큰 인증 수준(an 클레임)을 검증하지 않아, 별도 리포트 「authTicket 회원번호 미검증으로 임의 회원 인증 세션 탈취」에서 발견된 객체 수준 인가 결함으로 획득한 an=0(익명) 상태 세션으로도 호출이 가능하다. 결과 응답에는 피해자의 **주민번호 기반 연계정보(encCI), 전화번호, 생년월일, 성별, 실명**이 평문으로 포함된다.

주민번호 기반 연계정보(CI)는 한국 주민등록번호를 기반으로 산출한 연계정보로, 금융·통신·공공 서비스에서 동일인 확인에 사용되는 민감한 개인식별정보다. 개인정보보호법상 고유식별정보에 준하는 보호 대상이며, 실명·전화번호·생년월일과 함께 노출되어 피해자 개인정보가 광범위하게 수집된다.

취약한 소프트웨어: U.STRA Node Framework 3.2.1-stable (Nuxt.js 단일 페이지 애플리케이션), GS ALL 멤버십 서비스
취약점 발생 환경: 프로덕션 서버, 인증 전 접근 가능

### 취약점 상세 설명
#### 취약점 발견 방법
authTicket 객체 수준 인가 결함(별도 리포트) 분석 중 획득된 세션으로 접근 가능한 엔드포인트를 조사하던 중, 비밀번호 찾기 흐름의 `certEffecCheck` 엔드포인트가 an=0(익명) 세션으로 200 응답을 반환하는 것을 발견하였다. 응답에 주민번호 기반 연계정보(encCI)와 전화번호가 평문으로 포함된 것을 확인하여 취약점 존재를 검증하였다.

#### 취약점 발생 원인
**[근본 원인] U.STRA Framework 프로덕션 환경 인증 미들웨어 부재**

- Nitro 서버 라우트에 전역 인증 미들웨어 없음 (상세: 별도 리포트 「authTicket 회원번호 미검증」 참조)
- 결과: 각 엔드포인트가 개별적으로 인증 수준을 검증해야 하며, 미구현 시 an=0 세션으로 접근 허용

**[취약점] certEffecCheck 인증 수준 미검증**

- 발생 위치: `POST /api/hub/member/certEffecCheck`
- 발생 원인: 서명 토큰의 an 클레임(인증 수준) 검증 없음 → an=0 익명 세션으로 200 응답 및 개인정보 반환

#### 2. authTicket 객체 수준 인가 결함 선결 수정 (긴급)
이 취약점의 공격 시작 지점은 authTicket 객체 수준 인가 결함이다. authTicket에서 회원번호 소유권 검증을 추가하면 피해자 세션 획득 경로 자체가 차단된다.

#### 3. nuxt.env.secret 클라이언트 번들 제거
익명 서명 토큰 위조에 사용되는 서명 비밀키(`nuxt.env.secret`)를 클라이언트 번들에서 제거한다.

#### 4. certEffecCheck 응답 최소화
encCI(주민번호 기반 연계정보)를 응답에 포함할 절대적 필요가 없다면 제거한다. 포함이 필요하다면 인증된 세션에서만 반환하도록 한다.
