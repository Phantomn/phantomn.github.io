## 취약점 설명 및 조치 방안
### 취약점 개요
`PUT /api/hub/member/management/password-modify`가 인증 없이 호출 가능하며, 현재 비밀번호 검증도 없다. 통합회원번호(grmMbrNo)와 새 비밀번호만으로 임의 GS ALL 회원의 비밀번호를 즉시 변경할 수 있다. 별도로 발견한 authTicket 객체 수준 인가 결함(별도 리포트 참조)과 연계하면 회원번호 정수 1개만으로 3단계 만에 GS ALL 전체 회원의 계정을 완전히 탈취할 수 있다.

취약한 소프트웨어: U.STRA Node Framework 3.2.1-stable (Nuxt.js 단일 페이지 애플리케이션), GS ALL 멤버십 서비스
취약점 발생 환경: 프로덕션 서버, 인증 없이 접근 가능

### 조치 방안
#### 1. password-modify 서버 측 인증 검증 추가 (긴급)

`PUT /api/hub/member/management/password-modify`에서 요청자 서명 토큰이 실제 인증된 사용자임을 검증한다.

```javascript
// Nitro 엔드포인트 수정 예시
export default defineEventHandler(async (event) => {
  const auth = event.context.auth;

  // 1. 인증된 사용자 검증 (an=0 익명 요청 거부)
  if (!auth?.sub || auth.an === 0) {
    throw createError({ statusCode: 401, message: 'Authentication required' });
  }

  const body = await readBody(event);

  // 2. 서명 토큰의 통합회원번호와 변경 대상 통합회원번호 일치 검증
  if (auth.sub !== body.grmMbrNo) {
    throw createError({ statusCode: 403, message: 'Forbidden' });
  }

  // 3. 현재 비밀번호 검증 추가
  const isValid = await verifyCurrentPassword(body.grmMbrNo, body.currentPasswd);
  if (!isValid) {
    throw createError({ statusCode: 400, message: 'Current password is incorrect' });
  }

  // 비밀번호 변경 진행
  ...
});
```

#### 2. 현재 비밀번호 또는 일회용 비밀번호 인증 필수
비밀번호 변경 요청 시 `currentPasswd` 항목을 필수로 추가하고 서버에서 검증한다. 소셜 로그인(비밀번호 미설정) 계정의 경우 이메일·문자 일회용 비밀번호 인증을 대체 수단으로 요구한다.

#### 3. authTicket 객체 수준 인가 결함 선결 수정
이 취약점의 공격 단계는 authTicket 객체 수준 인가 결함(별도 리포트)에 의존한다. authTicket에서 회원번호 소유권 검증을 추가하면 통합회원번호 획득 경로가 차단되어 공격 단계가 1단계에서 중단된다. 두 취약점 모두 수정되어야 완전한 방어가 된다.

#### 4. nuxt.env.secret 클라이언트 번들 제거
익명 서명 토큰 위조에 사용되는 `nuxt.env.secret`을 클라이언트 번들에서 제거한다. 이 값이 제거되면 공격자가 서버 신뢰 서명 토큰을 위조할 수 없어 공격 진입점 자체가 차단된다.
